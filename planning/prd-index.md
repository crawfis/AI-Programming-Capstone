# PRD Index

*This file is created after kickoff, when you refine your project brief into an executable PRD.*

After kickoff, turn your project brief into a full Product Requirements Document. Say: "Help me refine my project brief into an executable PRD"

(The capstone-kickoff skill will guide you to this step automatically.)

This index will contain a summary of each feature spec, with pointers to individual files in `planning/prd/`. Each feature gets its own file with acceptance criteria that pass the Two-Developer Test: two developers reading the same spec should build the same thing.

See [Spec Quality Reference](../docs/spec-quality.md) for what makes a spec executable. See [Sample PRD](../docs/sample-prd.md) for a complete example.
